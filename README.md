## Discord Bot の作成と招待

### 1. アプリケーションを作成する

1. [Discord Developer Portal](https://discord.com/developers/applications) にアクセスしてログイン
2. 右上の **「New Application」** をクリック
3. 任意の名前（例: `UniteChan`）を入力して **「Create」**

---

### 2. Bot を作成する

1. 左メニューの **「Bot」** をクリック
2. **「Reset Token」** をクリックしてトークンを生成・コピー（後で `.env` に貼る）

> ⚠️ トークンは他人に見せないこと。漏れたらすぐ「Reset Token」で再発行する。

---

### 3. Privileged Gateway Intents を有効にする

Bot ページの **「Privileged Gateway Intents」** セクションで以下を **ON** にして **「Save Changes」**：

| Intent | 説明 | 要否 |
|--------|------|------|
| **Server Members Intent** | メンバー情報の取得・キャッシュ | ✅ 必須 |
| **Message Content Intent** | メッセージ内容の読み取り | ✅ 必須 |

> Presence Intent は不要です。

---

### 4. Bot をサーバーに招待する

1. 左メニューの **「OAuth2」** → **「URL Generator」** を開く
2. **Scopes** で以下を選択：
   - ✅ `bot`
   - ✅ `applications.commands`
3. **Bot Permissions** で以下を選択：

| 権限 | 用途 |
|------|------|
| **Send Messages** | チーム分け結果・戦績の送信 |
| **Embed Links** | Embed（カード形式）メッセージの送信 |
| **Add Reactions** | 🇦 🇧 🎙️ 🔄 🔁 リアクションの自動追加 |
| **Read Message History** | リアクション処理のためにメッセージを参照 |
| **Move Members** | 🎙️ リアクション・`/split move` でVCメンバーを移動 |
| **Attach Files** | `/stats export` で JSON ファイルを送信 |

4. ページ下部に生成された URL をブラウザで開き、招待したいサーバーを選んで **「認証」**

---

### 5. スラッシュコマンドを同期する

Bot 起動後、コマンドが表示されない場合はサーバーで `/sync` を実行する（Administrator 権限が必要）。

## セットアップ

### 必要な環境

- Python **3.11** 以上

### インストール

```bash
git clone https://github.com/LaceCelestia/UniteChan.git
cd UniteChan
pip install -e .
```

### 環境変数の設定

プロジェクトルートに `.env` ファイルを作成：

```env
DISCORD_TOKEN=ここにBotトークンを貼る
```

## 起動方法

### Windows（推奨）

```bat
run.bat
```

クラッシュ時に 5 秒後に自動で再起動します。

### 直接実行

```bash
python -m unitechan.app.bot
```

## コマンド一覧

★ = 管理者専用（Administrator / Manage Guild / Manage Roles いずれかが必要）

### ロビー

| コマンド | 説明 |
|---|---|
| `/join` | 自分がロビーに参加 |
| `/join <メンバー>` | 指定メンバーをロビーに追加 ★ |
| `/leave` | ロビーから抜ける |
| `/rank` | 自分のランクを登録・変更 |
| `/lobby` | 現在のロビーメンバーを表示 |
| `/lobby_collect` | 参加中のVCメンバーをロビーに一括登録 ★ |
| `/lobby_clear` | ロビーを全員解散 ★ |
| `/kick <メンバー>` | メンバーをロビーから追放 ★ |
| `/name [name] [member]` | ボット上の表示名を変更（省略でリセット、member 指定は ★） |

### チーム分け

| コマンド | 説明 |
|---|---|
| `/split run` | チーム分けを実行（コードは `/config split` で設定） |
| `/split test [モード]` | デモデータでチーム分けをテスト ★ |
| `/split move [channel_a] [channel_b]` | チーム分け結果に従いVCにメンバーを移動 ★ |
| `/split prev` | 過去5試合分の戦績を後から記録（各メッセージに 🇦/🇧） |

> **`/split run` 後のリアクション操作**
>
> | リアクション | 動作 |
> |---|---|
> | 🇦 / 🇧 | そのチームの勝利として試合結果を記録 |
> | 🎙️ | `/config vc` のVCに全員を一括移動 ★（移動後は 🎙️ と 🔄 が消える） |
> | 🔄 | 同じ設定でリロール（メッセージを更新） ★ |
>
> 🇦/🇧 で記録後の勝利メッセージ：
>
> | リアクション | 動作 |
> |---|---|
> | 🔁 | 同じメンバーで再戦の結果を続けて記録 |

### バン

| コマンド | 説明 |
|---|---|
| `/ban add <ポケモン>` | ポケモンをバン ★ |
| `/ban remove <ポケモン>` | バンを解除 ★ |
| `/ban list` | バン中ポケモン一覧を表示 |
| `/ban clear` | 全バンを解除 ★ |

### 戦績

| コマンド | 説明 |
|---|---|
| `/result <team>` | 試合結果を手動で記録（Team A / Team B） |
| `/result_undo` | 直前の試合結果記録を取り消し ★ |
| `/stats show [member] [period]` | 戦績ランキングまたは個人の戦績を表示（period: 通算 / 当日） |
| `/stats reset` | 通算・当日の戦績をすべてリセット ★ |
| `/stats export` | 戦績データをJSONファイルとして出力 ★ |
| `/stats import <file>` | 別サーバーからエクスポートした戦績をマージ ★ |

> 当日戦績は **JST 05:00** にリセットされます。

### 分離設定

| コマンド | 説明 |
|---|---|
| `/separate add <member1> <member2>` | 2人を必ず別チームに設定 ★ |
| `/separate remove <member1> <member2>` | 分離設定を解除 ★ |
| `/separate list` | 分離設定一覧を表示 |
| `/separate clear` | 全分離設定を解除 ★ |

### 設定

| コマンド | 説明 |
|---|---|
| `/config split <コード>` | デフォルトのチーム分けコードを設定 ★ |
| `/config role_balance` | ロール構成を設定（ATK/ALL/SPD/DEF/SUP 各人数） ★ |
| `/config avoid <回数>` | 連続ロール回避の回数を設定（0〜5） |
| `/config vc <team> <channel>` | 🎙️ リアクション・`/split move` で使うデフォルトVCを設定 ★ |
| `/config start_announce <minutes>` | VC移動後にスタート時刻を告知する分数（0でOFF、最大60） ★ |
| `/config show` | 現在の設定を表示 |
| `/config reset` | 設定をリセット ★ |

## チーム分けコード

`/config split <コード>` でサーバーごとに保存できます。5桁の数字で指定します。

```
[a][b][c][d][e]
 │  │  │  │  └─ e: チーム間ポケモン重複
 │  │  │  └──── d: 連続ロール回避
 │  │  └─────── c: ポケモン割当
 │  └────────── b: ロール割当
 └───────────── a: バランス方式
```

| 桁 | 意味 | 値 |
|---|---|---|
| **a** | バランス方式 | `0` = ランダム / `1` = ランクで均等 / `2` = 通算勝率で均等 / `3` = 当日勝率で均等 |
| **b** | ロール割当 | `0` = なし / `1` = 自動（各1枚） / `2` = `/config role_balance` の設定値 |
| **c** | ポケモン割当 | `0` = なし / `1` = 個人割当（1人1匹） / `2` = チーム割当（チームに5ロール各1匹） |
| **d** | 連続ロール回避 | `0` = OFF / `1` = ON（`/config avoid` の回数を使用） |
| **e** | チーム間重複許可 | `0` = 禁止（両チームで同ポケモン不可） / `1` = 許可 |

### 設定例

| コード | 内容 |
|---|---|
| `00000` | 完全ランダム |
| `10000` | ランクバランスのみ |
| `11100` | ランクバランス + ロール自動割当 + ポケモン個人割当 |
| `12110` | ランクバランス + ロール設定値 + ポケモン個人割当 + 連続回避 |
| `20000` | 通算勝率バランス |
| `21100` | 通算勝率バランス + ロール自動 + ポケモン個人割当 |
| `30000` | 当日勝率バランス（05:00リセット） |

> **バランス方式 a=1/2/3 のとき**：勝率・ランク差の均等化を最優先にしつつ、同じバランスのチームが複数ある場合はなるべく被りが少ない組み合わせを選びます。
>
> **a=0（ランダム）のとき**：同じチームになった累積回数が少ない組み合わせを優先します。

## VC移動の流れ

### 基本の流れ

1. **ロビーを組む** — `/lobby_collect` でVCメンバーを一括登録、または `/join` で個別に参加
2. **チーム分け** — `/split run` でチーム分けを実行
3. **VC移動** — 🎙️ リアクション（または `/split move`）で Team A / Team B のVCに自動振り分け
4. **結果記録** — 試合後に 🇦 または 🇧 をリアクションで押して戦績を記録

### スタート告知

`/config start_announce <minutes>` を設定すると、VC移動後に `⏰ HH:MM スタートです！` を自動送信します。

### 戦績を後から記録する

`/split prev` で直近5試合分のチーム構成が新しい順に表示されます。各メッセージの 🇦/🇧 を押すだけで後から記録できます（直近1試合のみ 🔁 再戦ボタンあり）。

### 11人以上のとき

10人を超えた分はランダムに観戦者として選出されます。連続観戦を避けつつ、観戦回数が少ない人を優先します。

## データファイル

`data/` フォルダに各種状態が自動保存されます。削除するとリセットされます。

| ファイル | 内容 |
|---|---|
| `data/lobby_state.json` | ロビーメンバー・ランク・表示名エイリアス |
| `data/config_state.json` | サーバー設定・バン中ポケモン・分離ペア・VC設定 |
| `data/stats_state.json` | 通算/当日戦績・試合履歴・ペア累積履歴・観戦履歴 |
| `data/pokemon_list.yaml` | ポケモンリスト（手動編集可） |
